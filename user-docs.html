Installation

  • Ubuntu
  • Mac OS/X
  • Windows

The easiest way to install Toothless is to run the install script in a terminal:

$ wget --quiet -O - https://raw.githubusercontent.com/wwwtyro/toothless/master/install.sh | sudo sh

If you'd like to do this manually yourself, follow the steps in the installation script.

Get audio working in Ubuntu Linux

Audio for X11 apps may not work out of the box on Ubuntu. To get it working, open the paprefs application (installed as a dependency of Toothless), click the Network Server tab, and enable Enable network access to local sound devices.

Launching an application

Launching an application with Toothless is as simple as clicking a link. After you've got it installed, click this link to launch the Gimp image editor, for example. The first time you launch an application, it will need to download and install the app. Later launches of the same application will be speedier.

Using the control panel

Toothless comes with a control panel that you can use by going to http://locahost:9887 in your browser. You can use this control panel to install, uninstall, start, and stop applications.

  1. The quicklaunch bar can be used to install and launch an application directly from the control panel. Type the name of the application repository into the bar and hit enter. If the application is not yet installed, it will be installed before launching it.
  2. The installed apps tab displays the applications that are currently installed.
  3. App launch links will launch an application when clicked.
  4. Uninstall app buttons will uninstall an application and all of its associated data.
  1. The running apps tab displays the applications that are currently running.
  2. App stop buttons will attempt to shut down an application gracefully.
  3. App kill buttons will immediately stop an application, with a risk of data loss.

Application file access

  • Ubuntu
  • Mac OS/X
  • Windows

Files are located in the /opt/toothless/app-data directory. There you'll find a directory for each application, as well as a shared path for data that is to be shared between applications.
